# RACKP Protocol Test Vectors

Published known-answer vectors for the protocol's cryptographic conventions.
Two independent implementations that both reproduce these vectors will produce
mutually verifiable hashes and signatures; without them, a mismatch in any of
the three layers below surfaces only as an opaque "signature verification
failed" at first contact.

**The secret keys in these files are deliberately public and burned.** They
exist only so implementations can compare intermediate values. Never use them
as a real terminal identity; a Keeper SHOULD reject registration of these keys.

## Conventions pinned by these vectors

1. **Canonicalization** — RFC 8785 (JCS). The canonical UTF-8 byte string is
   the input to both hashing and signing. (`jcs.json`)
2. **`data_hash`** — SHA-256 (lowercase hex) over the JCS canonicalization of
   the anchored payload. (`claim_anchor.json`)
3. **Signatures** — Ed25519 over the JCS canonicalization of the message body
   (all fields except `signature`), encoded as **unpadded Base64url** (no `=`
   padding). Public keys use the same encoding. (`signature.json`,
   `claim_anchor.json`)

## Files

| File | Covers |
|---|---|
| `jcs.json` | Key ordering (UTF-16 code units), escapes, non-ASCII, numbers, empty containers |
| `signature.json` | Key derivation, body canonicalization, signature bytes and encoding |
| `claim_anchor.json` | The full pipeline end to end, including every intermediate value |

## How to use

For each vector, feed `input` / `message_body` / `anchored_payload` to your
implementation and compare your output against the published intermediate and
final values. Test each stage separately — a canonicalization mismatch is
otherwise indistinguishable from a signing bug.

## Provenance

Generated by a clean-room Claimant implementation built solely from this
repository's specification documents (`tools/generate_vectors.py` in that
project). Its Ed25519 implementation is validated against the official
RFC 8032 Section 7.1 vectors (TEST 1–3). Regeneration is deterministic.
